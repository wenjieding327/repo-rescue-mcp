# 产品定位与履历表述

## 与通用 AI 的区别

ChatGPT 擅长解释、生成和建议修改；Codex 能直接读取项目、修改代码并运行工程任务。RepoRescue 不应宣称“比它们更会写代码”，而应把这种能力封装为学生无需懂命令、无需反复写提示词也能完成的固定救援流程。

真正差异：

- 自动识别该运行什么
- 兼容片段、报错、文件和仓库
- 强制显示修改前失败、修改后成功
- 区分推测、运行、指定测试、官方 Demo 和论文指标
- 输出完整代码、Diff、环境和复现报告
- 用零基础语言解释

如果只能解释和生成建议，它就是通用 AI 换皮；只有“执行验证 + 证据分级 + 科研复现”形成闭环，产品才成立。

## 真实用户

- 编程初学者和课程作业用户
- 接手陌生 GitHub 项目的本科生、研究生
- 需要批量复现论文的实验室
- AI 竞赛和课程项目参与者
- 导师、科研助理、审稿人
- 评估开源技术能否接入的企业团队

## 价值示例

- 一段代码：定位第 27 行类型错误，给最小 Diff，并证明修改后测试通过。
- GitHub 项目：发现 Python、CUDA、PyTorch 冲突，生成兼容环境并跑通官方示例。
- 陌生项目：输出文件职责、执行顺序和阅读路线。
- 结果错误：检查数据预处理、标签、随机种子和参数，重新运行并比较指标。

## 简历/GitHub 表述

> RepoRescue 是一套面向学生与研究人员的可验证代码救援和科研复现 Agent。它将生成式代码修复与隔离执行、前后对照测试、仓库固定及分级证据结合，支持从 Python 片段到公开 GitHub 项目的诊断—修复—验证闭环，并可通过 MCP/Skill 在多种 Agent 平台复用。

可量化成果必须以仓库测试和真实平台记录为准，不填写未经测量的成功率或节省时间。

## 比赛表达

一句话：

> 别的助手告诉你“可能怎么改”，RepoRescue 给你可复制的代码，并用同一组测试证明它是否真的修好了。

复赛故事：

> 原本需要两天配置的科研项目，RepoRescue 在受控环境中定位依赖、路径和文档问题，自动修复可授权部分，完成官方 Demo，并生成可核验的复现报告。
